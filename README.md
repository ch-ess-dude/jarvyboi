## First-time setup

You need Flutter в‰Ґ 3.22 and Dart в‰Ґ 3.3.

```bash
cd jarvy
chmod +x setup.sh
./setup.sh
```

What `setup.sh` does:
1. `flutter pub get` вЂ” installs all packages
2. `dart run build_runner build` вЂ” generates `lib/database/database.g.dart` (Drift schema)
3. `flutter run` вЂ” launches the app

**Important:** `database.g.dart` is generated code and is gitignored. You must run `build_runner` before the project will compile.


## Design system

Jarvy uses a **register-based** design system: each screen has a distinct tonal world.

| Screen | Register | Background | Accent | Display font |
|---|---|---|---|---|
| Dashboard | `daily` | `oklch(14% 0.008 60)` | Amber | Cormorant Garamond |
| Do | `ritual` | `oklch(13% 0.008 40)` | Copper | DM Serif Display |
| Log | `daily` | `oklch(14% 0.008 60)` | Amber | Cormorant Garamond |
| Body | `kinetic` | `oklch(10% 0.005 200)` | Electric green | Sora Bold |
| Build | `ambition` | `oklch(9% 0.005 250)` | White-gold | Playfair Display |
| Library | `literary` | `oklch(13% 0.006 80)` | Burgundy | Playfair Display |
| Settings | `utility` | `oklch(12% 0.002 250)` | Blue CTA | Geist |

Colors are computed from OKLCH via a port of the CSS color math, embedded in `jarvy_theme.dart`.

## Database schema

10 Drift tables: `daily_logs`, `habits`, `habit_completions`, `calorie_entries`, `workouts`, `projects`, `tasks`, `pins`, `interests`, `study_sessions`.

All local вЂ” no Firebase, no cloud, no auth. Data lives in `jarvy.db` in the app documents directory.

## Regenerating the database

If you change `database.dart` (add columns, new tables):

```bash
dart run build_runner build --delete-conflicting-outputs
```

For watching during development:

```bash
dart run build_runner watch
```
